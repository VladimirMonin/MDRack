## Installed package

The wheel and the sdist were each installed offline into separate clean temporary
virtual environments and exercised from a directory outside the repository with
isolated Python 3.11.15 mode. Both checks passed:

- package metadata/API version `0.3.0`;
- `import mdrack` and `import mdrack_core` from `site-packages`;
- all 18 documented compatibility modules and all 82 frozen module/symbol pairs;
- exact ordered `mdrack.public_api.__all__` (23 names) and
  `mdrack.public_api.models.__all__` (7 names);
- installed legacy CLI/`MDRackEngine` result parity for text, semantic, and hybrid;
- provider-free in-memory core index/delete;
- fresh candidate SQLite migration through `0007` and resource/unit/vector round trip;
- installed `mdrack status` emitted exactly one parseable JSON object;
- zero network attempts in the in-process smoke.

The reusable check is `scripts/check_installed_package.py`; its frozen inventory is
machine-checked against the compatibility registry.

## Evidence labels

- `unit/offline`: repository tests and deterministic fakes/fixtures.
- `local components`: real local SQLite and temporary filesystem exercised by the
  repository and installed-package checks.
- `installed package`: byte-identical wheel/sdist installed and exercised outside
  source imports.

The final repository quality run collected 1266 tests and passed all 1266 on
Python 3.11.15. Ruff, forbidden-dependency, core-boundary, working-diff, and
cached-diff checks also passed for the same S11R3 checkout. Relative links and
heading anchors resolved, and `mmdc` rendered all five current-architecture
Mermaid blocks successfully.

## Privacy and compatibility scope

The release gate covers the complete documented compatibility module/symbol inventory,
exact public exports, installed CLI/engine retrieval parity, CLI envelopes, engine/
image/resource DTOs, status/doctor/model/eval safe schemas, recovery/release diagnostic
allowlists, and recursive sentinels for query/content/path/root/endpoint/vector/
metadata/facet/provider-body/private-exception families. Public resource contracts
use logical identities and do not expose SQLite row IDs. The common release JSON
uses only the closed `{schema_version, generated_for, status, checks}` schema.

## Explicit non-claims

- No real source or vault corpus was read or mutated.
- No live LM Studio, OCR, caption, visual-vector, model load/inference/download, or
  other external provider request was executed.
- No Windows command or executable smoke was executed.
- No PostgreSQL/pgvector, audio/video ingestion, production reranking, or remote
  asset fetch is claimed.
- Deterministic fake and local SQLite evidence is not promoted to live or
  real-source evidence.
- No candidate cutover, cleanup, active-store mutation, staging, commit, or push
  was performed by this release gate.
