# MDRack 1.1 fixture freeze contract

Status: frozen public input bundle; no candidate evaluation output exists in this stage.

## Purpose

F1 freezes all public inputs before Q1 can inspect retrieval results. The bundle reuses the existing provider-free public corpus contract and adds an exact 1.1 execution config, privacy sentinels, capability/Definition-of-Done mapping, and a deterministic aggregate freeze manifest.

The normative freeze identity is `tests/evaluation/v1_1/freeze-manifest.json`. Its `contract_digest` is SHA-256 over canonical UTF-8 JSON with sorted keys, compact separators, one trailing LF, and the top-level `contract_digest` omitted. `input_sha256` then pins every corpus artifact, judgment document, schema, generator, config, v1.1 reader fixture, privacy sentinel file, and mapping file by repository-relative path and exact bytes.

## Frozen sources

- `tests/evaluation/corpus-v1/manifest.json`: 50 synthetic CC0 resources (20 Markdown, 10 image-text, 10 audio transcript, 10 video transcript; 5 videos include frame captions).
- `tests/evaluation/queries-v1/queries.json`: 170 graded cases (50 lexical, 50 semantic, 30 hybrid, 20 resource similarity, 20 timestamp).
- `tests/evaluation/benchmark-v1/manifest.json`: generated-data matrix contract only, not performance evidence.
- `tests/assets/v1_1/transcripts/`: deterministic four-format reader fixtures.
- `configs/eval-v11.toml`: fixed seed, metrics, ablations, weights, provisional thresholds, two-run policy, offline/static provider boundary, and explicit candidate-result prohibition.
- `tests/privacy/v1_1/sentinels.json`: synthetic leak sentinels for API, cache, CLI, evaluation, log, and provider surfaces.
- `tests/evaluation/v1_1/capability-dod-map.json`: all 19 reconciled capabilities and DoD sections 29.1–29.8; this is fixture routing, not completion evidence.

The public corpus validator remains authoritative for artifact hashes, opaque identities, provenance, SPDX allowlisting, PII review, timeline ownership, judgment linkage, roadmap counts, prohibited locators, and privacy-safe failures. The deterministic corpus generator is `tests/evaluation/generate_fixture.py`; the F1 aggregate generator/checker is `scripts/build_v11_test_assets.py`.

## Pre-result invariant

Q1 must execute `uv run python scripts/build_v11_test_assets.py --check` before creating or reading candidate results. Any byte or contract mismatch fails closed. Query labels or judgments may change only through an explicit version bump and a new freeze digest; they must never be edited in response to candidate output.

The freeze manifest and config contain no `results`, `metrics`, `scores`, or candidate output. Q1 owns disposable outputs outside the frozen input bundle and must publish only aggregate IDs, counts, digests, and sanitized observations.

## Verification

```bash
uv run python scripts/build_v11_test_assets.py --check
uv run pytest tests/evaluation/v1_1 tests/privacy/v1_1 tests/evaluation/test_public_contract.py
```

The generator is deterministic: running it twice must produce byte-identical `freeze-manifest.json`; `--check` performs no write and compares the expected bytes to the checked-in manifest.

## Evidence boundary and non-claims

This stage proves a deterministic, public, licensed, provider-free input freeze only. It does not prove candidate relevance, thresholds, application E2E behavior, latency, capacity, installed packages, a live provider, a private corpus, Windows, Python 3.12, visual similarity, acoustic similarity, or universal quality. No network, paid provider, private corpus, schema migration, destructive cleanup, tag, or publication is used by F1.
