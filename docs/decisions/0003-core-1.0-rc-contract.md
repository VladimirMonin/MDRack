# ADR-0003: MDRack core 1.0 release-candidate contract

- **Status:** Accepted for the core contract freeze
- **Date:** 2026-07-19
- **Contract version:** `1.0.0-rc.1`
- **Implementation baseline:** `afbea342fe8fb976ac787fa0c3fa884b469f6b36`
- **Applies to:** `mdrack_core` contract/domain/application/ports/observability surfaces
- **Supersedes:** no compatibility guarantee from ADR-0002; this ADR narrows and versions it

## Decision

`mdrack_core.CORE_CONTRACT_VERSION` is the reusable contract identity. It is
independent from the current combined `mdrack` distribution version; standalone
package metadata belongs to the later distribution slice.

The machine-enforced ordered export ledger is
`tests/core/test_contract_freeze.py`. It freezes the exact `__all__` tuples for:

- `mdrack_core`;
- `mdrack_core.domain`;
- `mdrack_core.application`;
- `mdrack_core.ports`.

The root surface contains domain records/constants, ports, safe observability records,
and `CORE_CONTRACT_VERSION`. Application services remain in
`mdrack_core.application`; fusion records remain there for v0.3 import compatibility.
No app DTO, Click type, SQLite type, filesystem path, provider type, or source-specific
locator helper is exported by core.

## Score and rank semantics

`ScoreKind` and `RankKind` make score/rank meaning explicit:

| Record/path | `score_kind` | `rank_kind` | Meaning |
|---|---|---|---|
| `RankedCandidate` | `adapter_raw` | `adapter_candidate` | Adapter-defined finite score and dense one-based rank after scope and before branch candidate limit |
| `FusedCandidate.score` | implicit `rrf` | n/a | Internal/export-retained weighted RRF score before final result projection |
| `RetrievalService` `SearchResultItem` | `rrf` | `result` | Weighted reciprocal-rank-fusion score and dense one-based final result rank |
| `ResourceDiscoveryService.similar` item | `adapter_raw` | `result` | Raw score from the selected vector metric and dense one-based similarity result rank |

A one-branch ordinary retrieval still returns an RRF score. Evidence candidates retain
the adapter score/rank semantics. Application compatibility mapping may continue to
project legacy text/semantic raw scores and hybrid RRF scores without changing its
public DTOs.

`SimilarityRequest.similarity_basis` is a required non-empty caller-owned value and is
copied to `SimilarityResult.similarity_basis`. Core treats the vocabulary as opaque; it
does not infer that a vector is textual, visual, or acoustic. The existing app resource
façade passes `legacy_unspecified` only as a compatibility bridge and does not add a new
public key in this slice. New public similarity surfaces must choose an explicit basis
before leaving that façade.

## Branch-local narrowing

`BranchScopeOverride` contains only these categorical fields:

- `resource_kinds`;
- `media_types`;
- `source_namespaces`;
- `representation_kinds`;
- `modalities`;
- `unit_kinds`.

For every field, an empty branch value means no additional restriction. Otherwise the
effective value is the intersection with the request-global value, or the branch value
when the global value is unbounded. A disjoint non-empty intersection skips that branch
without calling an adapter and without reporting adapter degradation.

`facets_any`, `facets_all`, and `facets_none` remain request-global and are forwarded
verbatim. Branch overrides cannot contain facets. This intentionally avoids treating
two `facets_any` OR clauses as a simple set intersection or expanding them into CNF.

## Locator envelope

`Locator(kind, payload)` is an opaque canonical envelope:

- `kind` is a non-empty open-vocabulary UTF-8 string;
- `payload` uses the bounded immutable JSON grammar and canonical serialization;
- core validates only envelope safety and persistence invariants;
- typed interval/path/frame/source semantics are validated by producer or compatibility
  packages before conversion to `Locator`;
- core never opens, resolves, stats, fetches, or interprets a locator.

## Deprecation ledger

| Surface | Introduced/frozen | State | Earliest removal gate |
|---|---:|---|---|
| Root/domain/ports/application ordered exports | `1.0.0-rc.1` | frozen RC | A later RC may change only with an ADR, exact export snapshot update, compatibility evidence, and migration note |
| `Locator(kind, payload)` | pre-RC; frozen `1.0.0-rc.1` | stable envelope | Breaking shape change requires core `2.0` after 1.x is final |
| `SearchResultItem.score_kind/rank_kind` | `1.0.0-rc.1` | stable RC | Same gate as ordered exports |
| `SimilarityRequest/Result.similarity_basis` | `1.0.0-rc.1` | stable RC | Same gate as ordered exports |
| `BranchScopeOverride` categorical fields | `1.0.0-rc.1` | stable RC | Adding facet algebra or widening ports requires a separate ADR and major-compatible migration |
| Fusion records exported by `mdrack_core.application` | pre-RC | compatibility-retained | No removal until importer inventory, deprecation notice, and installed consumer oracle pass |
| App resource façade `legacy_unspecified` basis bridge | app compatibility only | deprecated-by-design bridge | Replace in the later public similarity slice after explicit-basis CLI/Engine parity |

After a final `1.0.0` contract is declared, a public 1.x symbol is not removed or
semantically repurposed without prior deprecation in a released minor version and an
explicit migration/removal oracle. This RC does not itself claim final 1.0 stability.

## Conformance and compatibility evidence

The package-neutral consumer oracle is
`tests/external_core_consumer/test_memory_catalog_consumer.py`. It implements the
catalog/search ports and exercises indexing plus retrieval using only `mdrack_core`.
The shared Memory/SQLite suite remains adapter conformance evidence; this ADR does not
move or widen any port.

Existing app text/semantic/hybrid DTOs, CLI/Engine envelopes, stream behavior, and
source locators remain app-owned and unchanged. Core performs no stdout/stderr output.

## Rejected alternatives

- Full `SearchScope` per branch: rejected because facet OR algebra is not safely
  representable by naive tuple intersection.
- Adapter-side branch narrowing: rejected because all adapters must receive the same
  already-resolved effective scope.
- Inferring similarity basis from `space_id`, locator, or metadata: rejected because it
  couples opaque identities to producer semantics.
- Typed locators in core: rejected because media/source validation belongs outside the
  provider- and persistence-neutral envelope.

## Non-claims

This decision does not publish a standalone wheel, widen export/read ports, remove app
DTOs, add media/provider/storage implementation, change SQLite or migrations, prove an
installed package, or declare final 1.0 stability.
