# MDRack 1.1 transcript reader contract

Status: R1 application contract

## Boundary

The application owns four in-memory readers under
`mdrack.ingestion.transcripts`:

- `read_whisper_json()` for Whisper `segments` JSON;
- `read_vtt()` for WebVTT cues;
- `read_srt()` for SubRip cues;
- `read_timed_json()` for `mdrack.timed-transcript.v1`.

`read_transcript()` selects one of those readers from content. Detection never
uses a filename. Readers accept text, bytes, or (for JSON) an already decoded
mapping. They perform no filesystem or network I/O, do not call providers, do
not modify sources, and do not group atoms into passages.

Every successful reader returns `TranscriptReadResult`, containing the existing
`mdrack_media.TranscriptArtifact`, the detected format, the exact source SHA-256
fingerprint, and privacy-safe diagnostics.

## Required caller identity

The caller supplies:

- a canonical media `resource_id`;
- a typed `ProducerFingerprint` for the effective source producer/configuration;
- optional language metadata.

VTT and SRT readers never infer language from a filename. Generic timed JSON and
Whisper JSON use their declared language only when the caller did not override
it. Source-provided producer strings do not replace the caller-owned typed
producer fingerprint.

Representation and atom IDs use the existing `mdrack-media` identity functions.
Equivalent cue sequences with the same resource and producer therefore have the
same representation and atom IDs across all four formats.

## Timing and text normalization

- Whisper seconds are converted with decimal round-half-up to integer
  milliseconds. For example, `2.5005` becomes `2501`.
- VTT/SRT timestamps and generic timed JSON milliseconds are exact.
- Intervals must be non-negative, ordered, non-overlapping, and have
  `end_ms > start_ms`.
- Cue text is UTF-8, stripped, and Unicode whitespace is collapsed to one space.
- Subtitle cue markup is removed and HTML character references are decoded.
- Cue IDs, speaker, confidence, and timing precision are retained when valid.
- One accepted source segment/cue/atom becomes one `TimedTextAtom`; there is no
  passage chunking in R1.

The normalization policy has a deterministic `NormalizationFingerprint` shared
by all four readers. `duration_ms` is the last accepted atom end unless a valid,
larger source duration is supplied.

## Strict and lenient behavior

Strict mode is the default. The first invalid atom raises
`TranscriptReadError`. Its stable `code` and generic message never include raw
text, paths, metadata values, or exception bodies.

With `strict=False`, an invalid atom is skipped and a
`TranscriptDiagnostic(code, source_ordinal)` is returned. Accepted atom
ordinals and IDs are compacted to canonical `0..N-1` order. Diagnostics contain
only a category and ordinal.

Stable diagnostic categories include:

- `invalid_json`, `invalid_root`, `missing_segments`, `missing_atoms`;
- `invalid_atom`, `invalid_timing`, `empty_text`;
- `overlapping_interval`, `invalid_subtitle`;
- `unsupported_schema`, `unknown_format`.

Artifact metadata contains only the reader format, exact source fingerprint,
and diagnostic count. Reader code emits no logs.

## Generic timed JSON v1

The root schema must equal `mdrack.timed-transcript.v1` and `atoms` must be an
array. Each atom uses exact integer `start_ms` and `end_ms`, text, and optional
`atom_id`, `speaker`, `confidence`, and `timing_precision`. The external
`atom_id` is retained as source cue metadata; canonical media atom identity is
always generated by `mdrack-media`.

## Non-claims

This contract proves deterministic offline normalization only. It does not
claim Whisper execution, media decoding, provider quality, passage grouping,
search orchestration, live services, private corpus handling, Windows, or
Python 3.12 execution.
