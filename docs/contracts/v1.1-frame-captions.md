# Frame captions contract 1.1

Status: current application contract for the V1 offline video-composition slice.

## Scope

Frame captions are caller-supplied text observations. MDRack does not open a video, inspect frame bytes, call a caption provider, persist media binaries, or create image/vector modalities. Captions use the ordinary text embedding profile when embeddings are enabled.

## Frame manifest

`mdrack.frame-captions.v1` is strict JSON with these exact top-level keys:

- `schema`: `mdrack.frame-captions.v1`;
- `resource_id`: canonical media resource ID;
- `producer_fingerprint`: lowercase `sha256:` fingerprint;
- `normalization_fingerprint`: lowercase `sha256:` fingerprint or `null` for the application default;
- `metadata`: JSON object;
- `frames`: ordered array.

Each frame has exactly `frame_id`, `timestamp_ms`, `caption`, and `metadata`. The manifest `frame_id` is the caller's stable observation identity; MDRack derives the canonical unit ID from resource, producer, ordinal, timestamp, and observation identity. Timestamps are non-negative integer milliseconds. Captions are non-empty text. Duplicate observation identities and exact duplicate `(timestamp_ms, caption)` pairs fail closed before provider work or persistence at every application entry point.

Generic JSON metadata remains caller-controlled and is preserved. Dedicated extraction/provider fields named `frame_path` or `provider_payload` are forbidden at any nesting depth in artifact-level or per-frame metadata; manifests and direct application/Engine calls reject them with fixed payload-free errors before provider work or persistence. Portable resource `source_metadata` is unaffected by this frame-specific rule.

The reader performs no filesystem or network access beyond reading the explicitly selected manifest at the CLI boundary.

## Complete video replacement

`VideoCompositionService` accepts one canonical `TranscriptArtifact`, one `FrameCaptionArtifact`, a portable source locator, source metadata, and optional text embeddings. It builds separate `timed_passage`/`time_segment` and `frame_caption`/`frame` branches, then submits one `PreparedResourceBatch` to one `CoreIndexingService.index()` call. The SQLite adapter replaces the complete graph in one transaction.

Adding or removing frames therefore requires rebuilding the complete video graph from retained transcript and frame artifacts. Incremental unit patching is unsupported.

## Evidence

Transcript units use `time_segment` locators with `[start_ms, end_ms)`. Frame units use `video_frame` locators with `timestamp_ms` and the canonical frame unit ID. All units have modality `text`. No image embeddings, video embeddings, image bytes, or dedicated frame-path/provider-payload fields from frame artifacts are persisted.

## Privacy and non-claims

Raw transcript text, captions, metadata values, paths, locator payloads, vectors, and exceptions are not log fields. CLI errors use fixed payload-free categories. Offline fixture and fake-vector checks do not prove caption quality, visual similarity, live providers, FFmpeg, private corpora, Windows, or Python 3.12.
