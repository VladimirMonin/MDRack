# MDRack 1.1 application contract

Status: Frozen B0 contract for staged implementation
Baseline: `989aaaeaf029d7bdd2d9704ae578c9a19efe0f94`
Active plan: `active-plan-v4.md`, SHA-256
`f540960507b92a7d790aacae91b363f7884740978d792c6b3071f8b5a9821491`

## Purpose

This contract reconciles the proposed 1.1 roadmap with the checked-out owners.
It freezes only missing application seams. It does not rename the application
root, replay existing package work, or declare a final 1.0 release.

## Preserved foundations

The following are inputs to 1.1 application work, not implementation targets:

- `mdrack_core`: resource/representation/unit/vector/facet/locator records,
  complete-graph validation, scoped branches, grouping, weighted RRF, and
  textual similarity;
- `mdrack_media`: timed atoms/passages, transcript and frame artifacts, typed
  locators, deterministic grouping, builders, and whole-text aggregation;
- `mdrack_sqlite`: clean catalog schema `mdrack_sqlite_catalog_v1`, atomic
  replacement/deletion, metadata/facet round-trip, scoped pre-limit lexical and
  vector search;
- `mdrack.application.manifest`: strict prepared-resource manifest-v1 import;
- `MDRackEngine`, `PreparedResourceCatalog`, and the Click CLI as the current
  public application owners.

Package contract identities remain release candidates. B0 does not promote
`1.0.0-rc.1` to final 1.0.

The paths above are the checked-out `src/mdrack` owners. `app/src/mdrack` is an
illustration from the proposed roadmap and is not a valid implementation path.

## Cross-stage invariants

1. Core/media do not import source parsers, providers, storage, filesystem, or
   network code. `mdrack_sqlite` does not import `mdrack`.
2. SQLite is the only persistent database. Existing schema is used unless P1
   proves a blocker and separate schema/data authority is granted.
3. Application DTOs do not enter core. Provider calls finish before catalog
   mutation; catalog replacement remains complete and atomic.
4. Public identity uses logical IDs and portable locators, never new SQLite row
   IDs or absolute private paths.
5. Metadata is stored without loss for supported JSON types but influences
   retrieval only through an explicit projection.
6. Timed media remains text-first. Frame captions are text; similarity is
   textual; no image/audio/video embedding capability is implied.
7. Existing CLI envelopes and public imports are preserved. Additive 1.1
   surfaces are introduced only in their owning stage with parity and installed
   package evidence.
8. No default network, paid-provider, private-source, or destructive action is
   introduced.

## Compatibility and deferral

The exact B0 accounting is in
[`v1.1-compatibility-ledger.md`](../compatibility/v1.1-compatibility-ledger.md).
The entry evidence is in
[`v1.1-entry-gate.md`](../evidence/v1.1-entry-gate.md).

Deferred or excluded: arbitrary JSONPath, range filters, incremental media
patching, visual/acoustic embeddings, reranking, PostgreSQL/pgvector, daemon or
server mode, automatic source write-back, provider-backed quality claims, and
cache purge execution.
