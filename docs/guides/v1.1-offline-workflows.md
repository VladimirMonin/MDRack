# MDRack 1.1 offline workflows

MDRack 1.1 can index and retrieve already prepared text, metadata, transcript
passages, frame captions and ready vectors without contacting a provider. Audio and
video capabilities remain text-first: no image, audio or video embedding is implied.

## Check local readiness

```bash
mdrack --root ./vault status
mdrack --root ./vault doctor
```

Both commands are provider-free. `status` reports the selected generation, legacy
counts, profile identity and schema version without endpoint values. `doctor` checks
generation, FTS, embeddings and migrations. Neither command logs source paths,
content, metadata values, queries, vectors or provider responses.

## Move one complete prepared graph between catalogs

Create or obtain an existing clean `mdrack_sqlite_catalog_v1` catalog, then import a
manifest:

```bash
mdrack resource import prepared.json --catalog catalog.sqlite3
mdrack resource inspect resource-1 --catalog catalog.sqlite3
```

Export the same persisted graph through the unchanged manifest-v1 grammar:

```bash
mdrack resource export resource-1 \
  --catalog catalog.sqlite3 \
  --output resource-1.json
```

The output file intentionally contains the complete graph, including text, vectors,
locator payload and source metadata. Protect it like source data. The command refuses
to overwrite an existing file; stdout contains only identity, counts, byte size and a
digest. Reduced exports are explicit:

```bash
mdrack resource export resource-1 --catalog catalog.sqlite3 \
  --output resource-1-redacted.json --redact-source-metadata
mdrack resource export resource-1 --catalog catalog.sqlite3 \
  --output resource-1-lexical.json --no-vectors
```

A reduction that leaves any unit with neither text nor a vector fails instead of
writing an invalid manifest. Re-import proves semantic portability:

```bash
mdrack resource import resource-1.json --catalog restored.sqlite3
mdrack resource inspect resource-1 --catalog restored.sqlite3
```

Embedded hosts use the same codec and active catalog:

```python
payload = engine.export_resource_manifest("resource-1")
result = other_engine.import_resource_manifest(payload)
```

## Metadata-only local work

Markdown frontmatter is normalized into source metadata and influences retrieval only
through configured projections. Exact metadata inspection and facets are intentional
payloads; ordinary search/status/doctor output remains redacted.

```bash
mdrack --root ./vault scan --provider fake
mdrack --root ./vault metadata show document-1
mdrack --root ./vault metadata facets --namespace status
mdrack --root ./vault search "architecture" --mode text --target resource \
  --meta '/status="ready"'
```

`fake` is deterministic offline orchestration evidence, not model-quality evidence.
Store-only metadata is not embedded by default.

## Timed transcripts and frame captions

Import already available transcript text without embeddings:

```bash
mdrack ingest transcript lecture.vtt \
  --format vtt --resource-id audio:lecture-1 --kind audio \
  --media-type audio/wav --namespace local --catalog catalog.sqlite3 \
  --no-embeddings
```

Search lexical timed passages and receive millisecond evidence:

```bash
mdrack search "event sourcing" --catalog catalog.sqlite3 --mode text
```

A prepared video graph may combine transcript passages and text frame captions in one
atomic replacement. `speech_first`, `balanced` and `frames_first` are explicit
resource-search presets; they do not inspect pixels or infer query intent.

```bash
mdrack search "diagram" --catalog catalog.sqlite3 --mode text \
  --target resource --preset frames_first
```

## Textual similarity and cache

Whole-resource similarity is explicitly based on stored text vectors. Ready vectors
can be imported in a manifest, so the lookup itself is provider-free:

```bash
mdrack resource similar whole-resource-unit-1 \
  --catalog catalog.sqlite3 --space-id text-space \
  --embedding-fingerprint text-space-v1 \
  --aggregation direct-text --basis textual-content
```

Provider artifacts, when produced elsewhere, use the immutable content-addressed
cache. Local inspection and verification do not call providers:

```bash
mdrack --root ./vault cache status
mdrack --root ./vault cache verify
```

Cache purge, live LM Studio/OpenAI-compatible providers, OpenRouter, FFmpeg, private
corpora, Windows and Python 3.12 are separate execution or approval boundaries. This
guide makes no claim for them.
