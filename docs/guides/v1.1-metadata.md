# Metadata in MDRack 1.1

MDRack keeps normalized source metadata on the resource and lets only configured
projections affect retrieval. Unknown fields are `store_only`: they remain available
through explicit inspection, but do not enter facets, FTS, body chunks, snippets, or
embedding inputs.

## Configure projections

```toml
[[metadata.projections]]
path = "/title"
mode = "canonical_title"

[[metadata.projections]]
path = "/tags"
mode = "facet_many"
namespace = "tag"

[[metadata.projections]]
path = "/status"
mode = "facet"
namespace = "status"

[[metadata.projections]]
path = "/aliases"
mode = "lexical_text"

[[metadata.projections]]
path = "/private_plugin_state"
mode = "store_only"
```

Paths use JSON Pointer escaping (`‾0` for `‾`, `‾1` for `/`). Facets accept JSON
scalars; `facet_many` accepts arrays of scalars. String, integer, float, boolean, and
null remain distinct exact values. Range, regex, arbitrary JSONPath, and object
flattening are not supported.

## Lexical metadata

Each resource with at least one non-empty `lexical_text` projection receives one
separate `metadata_text` representation and one `whole_resource` text unit. Values
follow projection order and source array order with stable de-duplication.

The metadata unit is indexed in FTS but receives no vector by default. Regular
unit-target search remains scoped to body `retrieval_text`/`text_chunk` units.
Resource-target text search combines a body branch (weight `1.0`) with the separate
metadata branch (default weight `0.2`). Setting `--metadata-weight 0` disables the
metadata branch.

```bash
mdrack search "rare alias" --mode text --target resource
mdrack search "architecture" --mode text --target resource --metadata-weight 0
```

## Exact filters

`--meta`, `--meta-any`, and `--meta-none` accept
`JSON_POINTER=JSON_SCALAR`. JSON strings therefore include quotes. `--tag` is a
convenience exact string filter for the `tag` namespace.

```bash
mdrack search "architecture" --mode text --target resource ¥
  --tag python ¥
  --meta '/status="ready"' ¥
  --meta-any '/priority=3' ¥
  --meta-none '/draft=true'
```

Filters are compiled through the same typed scalar codec used during ingestion and
are applied by the catalog before branch candidate limits. A path not configured as
`facet` or `facet_many` fails closed; MDRack never scans stored metadata JSON at query
time.

The embedded equivalents are `mdrack.public_api.models.MetadataFilter`,
`MetadataFilters`, `MDRackEngine.search_text(..., metadata_filters=...)`, and
`MDRackEngine.search_resources_text(...)`.

## Explicit inspection

```bash
mdrack metadata show <resource-id>
mdrack metadata facets [--namespace status]
mdrack metadata projection-check note.md
```

`metadata show` is the intentional surface that returns exact normalized source
metadata and decoded typed source facets. Search results, logs, diagnostics, status,
and ordinary resource inspection do not echo those values.

`metadata projection-check` reads one Markdown file without writing the catalog. It
reports canonical title, configured lexical/facet/store-only/ignored paths, safe
diagnostic categories/counts, and the projection policy fingerprint. Store-only
values are not emitted.

## Boundaries

This local/offline path does not call LM Studio or another provider, does not modify
source Markdown, and does not add a schema migration. It does not prove live provider,
private-corpus, Windows, or Python 3.12 behavior.
